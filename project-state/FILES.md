# Files

- `backend/modules/vip30.js` — VIP30 Kernmodul, Settings, Reward, Dry-Run, interne Channelpoints-/EventSub-Bridge, Live-Dry-Run-Diagnose.
- `config/vip30.json` — Seed/Fallback, DB bleibt primär.
- `docs/modules/vip30.md` — Modul-Doku.
- `project-state/CURRENT_STATUS.md` — aktueller Stand.
- `project-state/NEXT_STEPS.md` — nächste Schritte.
- `project-state/TODO.md` — offene Punkte.
- `project-state/CHANGELOG.md` — Änderungen.
- `project-state/FILES.md` — Dateiliste.

Es gibt kein separates VIP30-Bridge-Zusatzmodul. Die Bridge sitzt intern in `backend/modules/vip30.js`.
