# Channelpoints v0.6.0 — Media Execution Bridge

Interner Arbeitsstand: Medien-Ausführungsbrücke für Kanalpunkte.

## Ziel

Kanalpunkte-Rewards sollen Medien nicht nur speichern, sondern über dasselbe zentrale Pattern wie Commands ausführen:

```text
mediaId -> /api/sound/play Payload
```

## Ergebnis

- Modulversion `0.6.0`
- Build `media-execution-bridge`
- lokale Execute-/Check-Routen vorhanden
- keine Twitch-Schreibzugriffe
- keine Funktionalität entfernt
