## Wichtige Regeln

Arbeitsweise zwingend einhalten:

```text
- GitHub/dev ist Single Source of Truth.
- Echte Dateien pruefen, nicht raten.
- Keine Funktionalitaet entfernen.
- Keine Parallelstrukturen bauen, vorhandene Module/Helper nutzen.
- Vor Umsetzung Scope nennen und auf ausdrueckliches go warten.
- Pro Antwort maximal ein Befehlsblock.
- Vor Befehlen immer sagen: Wo ausfuehren, was macht der Befehl, wann stoppen, welche Ausgabe schicken.
- Keine langen unnoetigen Ausgaben anfordern.
- ZIPs mit echten Repo-Pfaden ab Repo-Root liefern.
- Kein Desktop als Standardpfad.
- Kein git add . verwenden.
- StepDone erst nach Einspielen/Deploy/Test.
```

Server-Regel ab RDAP7C1:

```text
Keine RDAP-Arbeitsordner, Deploy-Clones oder Backups mehr direkt in /root.
Deploy-/Test-Clones: /opt/stream-control-center/_deploy_tmp/
Runtime-/Temp:       /opt/stream-control-center/_runtime_tmp/
Backups:             /var/backups/stream-control-center/
```

Security-Regeln:

```text
- Keine Secrets ins Repo, Frontend oder Chat.
- Backend prueft Rechte; Frontend ist keine Sicherheitsentscheidung.
- Keine Schreibfunktionen ohne Permission/Lock/Audit.
- Keine produktiven Agent-Actions ohne Allowlist.
- Keine freie Shell-/Datei-/Prozesssteuerung.
- Rollen und Gruppen getrennt halten.
- sound_profi ist Gruppe/Marker, keine Rolle.
- VIP gibt keine Dashboard-Grundrechte.
```
