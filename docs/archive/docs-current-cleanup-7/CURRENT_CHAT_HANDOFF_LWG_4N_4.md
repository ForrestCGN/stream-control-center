## Neue Entscheidung

Die Loyalty-Oberfläche wird nicht weiter als einzelne, überladene Tab-/Formularseite geflickt.

Stattdessen wird die Zielstruktur als Loyalty-Zentrale festgelegt:

- Übersicht
- Punkteverwaltung
- Glücksrad
- Giveaways
- Raffles
- Texte
- Statistik
- Config
- Verlauf

## Commands

Commands werden nicht als eigener Loyalty-Hauptbereich geführt.

- Bearbeitung im zentralen Command-Editor.
- Loyalty darf Standard-Commands prüfen und fehlende per Knopfdruck anlegen.
- Bestehende Commands werden niemals überschrieben.

## Presets

- Globale Presets gehören UI-seitig unter Glücksrad.
- Giveaway-Glücksräder gehören nur ins jeweilige Giveaway.
- Der gemeinsame Editor wird weiterverwendet und kontextabhängig vereinfacht.
