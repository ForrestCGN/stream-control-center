# Loyalty Navigation Rework

## Ziel

Das Loyalty-System wird als nutzerfreundliche Zentrale strukturiert. Die aktuelle überladene Mischansicht soll schrittweise ersetzt werden.

## Hauptbereiche

- Übersicht
- Punkteverwaltung
- Glücksrad
- Giveaways
- Raffles
- Texte
- Statistik
- Config
- Verlauf

## Presets

Globale Presets werden im Bereich Glücksrad verwaltet. Giveaway-gebundene Glücksräder erscheinen nicht in der globalen Preset-Liste.

## Editor

Der Preset-/Glücksrad-Editor bleibt ein gemeinsamer Editor. Je nach Kontext werden unnötige Felder ausgeblendet.

## Commands

Commands werden im zentralen Command-Editor bearbeitet. Loyalty darf nur prüfen und fehlende Standard-Commands nach Bestätigung anlegen. Bestehende Commands werden nicht überschrieben.

## Texte

Texte werden über die zentrale Multi-Textverwaltung mit Modulauswahl verwaltet.

## Statistik

Statistik zeigt Auswertungen. Verlauf zeigt chronologische Ereignisse.
